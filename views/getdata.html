Get the data

The data provided by our team can be obtained in the following ways.

Via API

The prefix of API is: {{api_prefix}}

It is backed by Google App Engine's NDB. You can pass filtering rules via URL parameters. Filters are additive so that you can combine them in any ways that suits your application. API is suitable for application authors who don't want to host the backend database. We'll walk you through some samples.

Get all the options

The API response is composed of the following elements:

  • meta Meta data like execution time.
  • data A list of matched values.
  • optios Possible options to construct further filters.

Raw JSON RESTful Interface

The prefix of the interface is: {{raw_json_prefix}}

The layout is as follows:

.
├── A01
│   ├── sheet0
│   │   ├── table0.json
│   │   ├── table10.json
│   │   ├── table11.json
│   │   ├── table12.json
│   │   ├── table13.json
│   │   ├── table14.json
│   │   ├── table15.json
...

You can go to the URL prefix first and navigate through the HK census data set. Or you can use it as RESTful interface to query the raw JSON tables converted from Data.One's xlsx files.
